# 트리거 시나리오 회귀 방지 픽스처

> 회귀 사이클(하나 고치면 하나 깨짐) 방지용. **AI 발화 → 기대 트리거** 누적 기록.
> 패치 전 `node tests/run-triggers.mjs` 실행 → 전 케이스 통과 확인 후 푸시.
> 새 케이스 만나면 여기에 추가 → 다음 패치도 자동 보호.

---

## 사용법

```bash
node tests/run-triggers.mjs              # 전 케이스 실행
node tests/run-triggers.mjs --only=형태  # 특정 카테고리만
```

⚠️ **이 파일과 `js/ui.js` updateQuickFromText의 정규식이 어긋나면 의미 없음.**
ui.js 트리거 정규식을 수정하면 `tests/run-triggers.mjs` 의 정규식도 같이 수정.

## 시나리오

각 줄: `기대트리거 | "AI 발화" | 참고`

### 형태 (shape)
```
shape    | 드레스룸이군요! 혹시 일자형(한 벽면)인지, ㄱ자형(코너 두 벽)인지, 아니면 다른 형태로 생각하고 계세요? | 형태명 2개+ 나열 + 질문어
shape    | 어떤 형태로 생각하세요?                                         | 형태 표현
none     | ㄱ자형은 60만원대, ㄷ자형은 80만원대예요                        | 가격 설명 (질문어 없음 — 오발동 방지)
```

### 옵션 (option)
```
option   | 서랍장이나 선반 같은 옵션 생각 있으세요?                        | 옵션+질문어 어순 무관
option   | 거울장이랑 서랍장 추가하실래요?                                 | 품목명 2개+ 나열
option   | 혹시 원하시는 옵션 있으세요?                                    | 원하 + 옵션 어순 무관
option   | 추가 옵션은 필요하세요?                                         | 옵션.*추가/필요
none     | 이런 구성이 어떠세요?                                           | 구성 단독 (오발동 방지)
none     | 예상 구성이 어떠세요?                                           | 견적 구성 안내 (오발동 방지)
```

### 천장 (ceiling)
```
ceiling  | 천장 높이가 어떻게 되세요?                                      | 천장+어떻게/되세요
ceiling  | 천장 몇 mm예요?                                                 | 천장+몇 mm
ceiling  | 층고 얼마예요?                                                  | 층고+얼마
curtain  | 천장 높이 2400~2700mm군요. 커튼박스는 있으신가요?               | 천장 요약+커튼 질문 → 커튼이 이겨야
```

### 커튼박스 (curtain)
```
curtain  | 커튼박스는 있으신가요?                                          | 커튼박스 단독
```

### 치수 (dimension)
```
dimension | 각 면 길이가 어떻게 되세요?                                    | 면 길이
dimension | 치수 어떻게 되세요?                                            | 치수+어떻게
dimension | 벽 길이 알려주세요                                             | 벽 길이
budget    | 예산은 얼마 생각하세요?                                        | 예산 질문 → budget. 치수로 오발동 안 됨 동시 검증
```

### 설치 공간 (space)
```
space    | 어느 공간에 설치하실 거예요?                                    | 어느 공간
```

### 설치 지역 (region)
```
region   | 설치 지역 어디세요?                                             | 설치 지역
region   | 어디 사세요?                                                    | 어디 사세요
```

### 예산 (budget)
```
budget   | 예산은 얼마 생각하세요?                                         | 예산+얼마
budget   | 얼마쯤 생각하고 계세요?                                         | 얼마쯤
```

### 색상 (color)
```
colorNarrow | 블랙 계열이면 프레임은 블랙, 선반은 다크월넛이나 진그레이가 잘 어울려요. 둘 중에 어떤 느낌이 더 좋으세요? | 색상명 2개+ + 질문어
colorShelf  | 선반 색상은 어떤 걸로 하실까요?                              | 선반 색상은
colorFrame  | 프레임 색상은 어떤 걸로?                                     | 프레임 색상은
confirmQuote | 이런 구성으로 예상 견적 정리해드릴까요?\n- 기본 행거: 480,000원\n- 거울장: 169,000원\n- 프레임: 블랙 / 선반: 진그레이\n합계 817,000원 + 배송비 20,000원 별도 | 색상명 섞여있어도 견적 요약은 confirmQuote 우선
```

### 선반 단수 (shelfCount)
```
shelfCount | 선반 몇 단으로 하실까요?                                      | 선반 몇 단
```

### 동의·확인 (consent / confirmReceive)
```
consent        | 개인정보 수집 동의해 주시겠어요?                          | 개인정보 동의
confirmReceive | 위 내용이 맞으신가요? 확인해 주시면 접수해드릴게요         | 맞으신가요
```

### 도면 예시 (example)
```
example | 예상 도면 보여드릴까요?                                          | 도면+보여드릴까
example | 구성 예시 이미지 보여드릴까요?                                   | 구성예시+보여드릴까
example | 3D 구성 예시 보실래요?                                           | 3D+보실래요
none    | 본 견적은 도면 확정 전 예상 견적이며 실제 시공 시 변경될 수 있습니다 | 견적서 안내 문구 (오발동 방지)
none    | 도면 작업해서 보내드릴게요                                       | statement (오발동 방지)
```

### 시공 사례 (carousel)
```
carousel | 비슷한 사례 보여드릴게요                                        | 비슷한 사례
```

### ①②③ 선택지 (circled / shape 라우팅)
```
shape    | 작은방이군요! 행거를 어느 쪽에 설치하고 싶으세요?\n① 한쪽 벽만\n② 코너 두 벽\n③ 세 벽 다\n④ 마주보는 두 줄 | ①②③ 4개인데 형태 키워드 → shape 카드 라우팅
circled  | 어떻게 진행할까요?\n① 바로 견적 받기\n② 조금 더 상담 | 형태 키워드 없음 → 일반 circled
```

### 견적서 (quote — isQuote 분기)
```
quote | 확인했어요!\n\n[설치 공간]\n- 드레스룸 ㄷ자 3면\n[금액]\n- 합계: 143만원\n[안내]\n- 본 견적은 예상\n\n신규아파트이신가요? | [설치공간]+[금액]
```

## 추가 규칙

- **새 회귀 만나면 여기 추가** → 그 회귀가 다시 못 들어옴
- 케이스 추가 후 `node tests/run-triggers.mjs` 1회 실행 → 다 통과 확인 후 푸시
- 트리거 ID(`shape`, `option` 등)는 `tests/run-triggers.mjs` 스크립트의 매핑과 1:1
